# Development Rules

## General Development Standards
- Follow conventional commit format with scope: `type(scope): description`
- Maintain TypeScript best practices
- Keep comprehensive test coverage
- Use proper error handling patterns
- Follow the established Azure Storage patterns and conventions

## Code Quality
- Use proper TypeScript types and interfaces
- Implement comprehensive error handling
- Write clear and maintainable code
- Follow consistent naming conventions
- Add proper JSDoc comments for public APIs

## Testing
- Write unit tests for all public methods
- Use proper test coverage
- Test error scenarios and edge cases
- Mock Azure Storage dependencies appropriately

## Azure Storage Design
- Support multiple Azure Storage services (Blob, Table, Queue)
- Provide consistent storage interface
- Handle Azure-specific errors appropriately
- Implement proper connection management
- Support different authentication methods
- Provide proper TypeScript types for Azure Storage operations
